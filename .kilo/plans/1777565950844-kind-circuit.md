# Create Comic Feature Implementation Plan

## Summary
Update the comic creation flow to use `multipart/form-data` for consistency with the existing update endpoint, enabling proper file upload support for thumbnails.

## Current State Analysis

### Backend (comic.ts)
- **POST `/comics/`** (lines 191-286): Uses JSON body with `categories` and `genres` as JSON strings (parsed via `JSON.parse()`). Has `Type.Object` body schema without `isMultipart`.
- **PUT `/comics/:id`** (lines 287-397): Uses `multipart/form-data` with `categories` and `genres` as arrays. Has `isMultipart: true` and uses `t.Partial(t.Object({...}))` body schema.

### Frontend
- **Create page** (`create/page.tsx`): Uses `api.api.comics.post()` with JSON body, passing `categories` and `genres` as JSON strings via `JSON.stringify()`.
- **Edit page** (`edit/page.tsx`): Uses native `fetch` with `FormData`, passing `categories` and `genres` as comma-separated strings (e.g., `"Action,Fantasy"`).

### Key Differences
| Aspect | POST (Create) | PUT (Update) |
|--------|---------------|--------------|
| Content-Type | JSON | multipart/form-data |
| categories/genres format | JSON string (`"[\"Action\"]"`) | Comma-separated string (`"Action,Fantasy"`) |
| File upload via | Base64 in JSON | FormData File object |
| Body schema | `Type.Object({...})` | `t.Partial(t.Object({...}))` with `isMultipart` |
