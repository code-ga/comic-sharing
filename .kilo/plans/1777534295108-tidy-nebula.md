# Plan: Add thumbnail field to comic object and implement file upload

## Goal
Add a thumbnail field to the comic object in the database and implement file upload functionality using the existing @backend/src/utils/files.ts utility.

## Changes Needed

### 1. Database Schema Update
- Modify `backend/src/database/schema/comic.ts` to add a `thumbnail` field to the comics table
- The thumbnail field should store the URL of the uploaded image

### 2. Backend Route Updates
- Modify `backend/src/routes/comic.ts` to handle thumbnail upload during comic creation and updates
- Use the `uploadImages` function from `@backend/src/utils/files.ts` to upload thumbnail images
- Update the comic creation endpoint to accept and process thumbnail files
- Update the comic update endpoint to handle thumbnail changes
- Consider adding a dedicated endpoint for thumbnail upload if needed

### 3. Type Definitions
- Update TypeBox schemas in the comic routes to include the thumbnail field
- Ensure Drizzle types are updated to reflect the new thumbnail field

### 4. File Upload Implementation
- When creating/updating a comic, if a thumbnail file is provided:
  1. Use `uploadImages` from `@backend/src/utils/files.ts` to upload the file
  2. Store the returned URL in the comic's thumbnail field
- When deleting a comic, consider if we should also delete the thumbnail from CDN (using `removeImage`)

## Implementation Details

### Database Changes
Add to comics table in `backend/src/database/schema/comic.ts`:
```typescript
thumbnail: text("thumbnail"), // URL of the thumbnail image
```

### Route Changes
In `backend/src/routes/comic.ts`:
- POST `/` (create comic): Accept thumbnail file, upload it, store URL
- PUT `/:id` (update comic): Handle thumbnail updates
- Consider adding POST `/:id/thumbnail` endpoint for separate thumbnail upload

### File Upload Flow
1. Client sends multipart/form-data with comic data and thumbnail file
2. Backend extracts the file and calls `uploadImages([thumbnailFile])`
3. Backend stores the returned URL in the comic's thumbnail field
4. Return the comic object with thumbnail URL in response
