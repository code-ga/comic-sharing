# Comic Detail Page Implementation Plan

## Overview
Implement a comic detail page (`/comics/[comicId]`) that displays comic information and chapters. The page should have two modes:
- **Author mode**: Show comic management tools (edit comic, add chapter, edit/delete chapters)
- **Reader mode**: Show read-only comic information and chapter list

## Tech Stack
- Next.js App Router (dynamic route)
- React Query for data fetching
- Tailwind CSS (Dracula theme)
- Type-safe API calls via Eden treaty client

## Implementation Steps

### 1. Create Dynamic Route Structure
- File: `frontend/app/(protected)/comics/[comicId]/page.tsx`
- Fetch comic details using `/comics/:id` endpoint (includes chapters)
- Determine user authorization (is the logged-in user the author?)

### 2. Page Layout & Components

#### A. Hero Section (Top)
- Comic thumbnail (large display)
- Comic title and description
- Categories and genres badges
- Edit button (visible to authors only)
- Delete button (visible to authors only)

#### B. Chapters Section
- Chapter list with sortable index
- Each chapter shows: title, index number
- Author actions per chapter: edit pencil, delete trash
- "Add Chapter" button (visible to authors only)

#### C. Action Buttons
- Authors see: Edit Comic, Add Chapter, Delete Comic
- Readers see: No editing options

### 3. Data Flow
- Use `useQuery` to fetch comic details from `api.api.comics({ id: comicId }).get()`
- The endpoint returns comic with `chapters` included (already implemented in backend)
- Use `useMutation` for delete operations (comic/chapter)
- Use `useQueryClient` for cache invalidation

### 4. UI Components Reuse
- `ComicCard` component (existing) - not used directly but similar styling
- `ChapterList` component (existing) - adapts for reader view
- Create new `ComicDetailHeader` component for hero section
- Create new `ChapterActions` component for chapter CRUD buttons

### 5. Authorization Logic
- Get current user from `useAuth()` hook
- Compare `user.id` with `comic.authorId`
- Show/hide edit controls based on authorization

### 6. Error & Loading States
- Loading: Skeleton placeholders for hero section and chapters
- Error: Display error message with retry option
- Empty chapters: "No chapters yet" message for readers, "Add your first chapter" for authors

### 7. Navigation & Links
- Back button to return to previous page
- Links to edit comic page: `/comics/[comicId]/edit`
- Links to add chapter: `/comics/[comicId]/chapters/create`
- Links to edit chapter: `/comics/[comicId]/chapters/[chapterId]/edit`

## Backend Endpoints Used
- `GET /comics/:id` - Fetch comic with chapters (already exists)
- `DELETE /comics/:id` - Delete comic (already exists)
- `DELETE /chapters/:id` - Delete chapter (already exists)

## Frontend Files to Create/Modify

### Create:
1. `frontend/app/(protected)/comics/[comicId]/page.tsx` - Main detail page
2. `frontend/components/ComicDetailHeader.tsx` - Hero section component
3. `frontend/components/ChapterActionButtons.tsx` - Inline chapter action buttons

### Potentially Modify:
1. `frontend/components/ChapterList.tsx` - Add prop to toggle edit controls visibility

## Data Structures

### Comic (from backend)
```typescript
{
  id: number;
  title: string;
  description: string;
  thumbnail: string | null;
  categories: string[];
  genres: string[];
  authorId: string;
  createdAt: string;
  updatedAt: string;
  chapters: Chapter[];
}
```

### Chapter (from backend)
```typescript
{
  id: number;
  title: string;
  index: number;
  comicId: number;
  authorId: string;
  createdAt: string;
  updatedAt: string;
}
```

## Styling Guidelines
- Follow Dracula theme (purple #bd93f9, pink #ff79c6, cyan #8be9fd)
- Glassmorphism cards with rounded corners
- Hover effects on interactive elements
- Consistent spacing and typography
- Responsive design (mobile-first)

## Edge Cases
- Comic not found (404)
- User not authorized (403) - redirect to login or show error
- No chapters for this comic
- Large chapter lists (scrollable container)
- Deleting comic with chapters (confirmation dialog)
