# Plan: Extract Layout Components

## Objective
Extract navbar/header and footer components from frontend/app layouts into separate reusable components in the `frontend/components/` directory, replacing the inline code in layout files.

## Current State
- `frontend/app/layout.tsx` - Public layout with header (logo, nav, signin/register) and footer
- `frontend/app/(protected)/layout.tsx` - Protected layout with auth-aware header (user greeting, signout)
- `frontend/components/` - Contains ComicCard, ChapterList, auth form components

## Plan Steps

### 1. Create `Navbar` component (`frontend/components/Navbar.tsx`)
- Extracted from public layout header
- Contains: Logo/brand link, navigation (Sign in / Get Started buttons)
- Responsive (hidden on mobile for md+)
- Uses Next.js Link
- Accepts props if needed for variants

### 2. Create `ProtectedNavbar` component (`frontend/components/ProtectedNavbar.tsx`)
- Extracted from protected layout header  
- Contains: Logo, user greeting, signout button OR signin/register links (auth-aware)
- Uses `useAuth` hook for session state
- Handles signout logic (currently reloads page)
- Loading state placeholder

### 3. Create `Footer` component (`frontend/components/Footer.tsx`)
- Extracted from public layout footer
- Contains: Copyright, legal links (About, Privacy, Terms, Contact)
- Simple, reusable

### 4. Update `frontend/app/layout.tsx`
- Remove inline header JSX
- Remove inline footer JSX
- Import and use `Navbar` and `Footer` components
- Remove now-unused font imports if not needed by components

### 5. Update `frontend/app/(protected)/layout.tsx`
- Remove inline header JSX
- Import and use `ProtectedNavbar` component
- Pass auth state/handlers or let component handle internally
- Remove now-unused font imports if not needed

### 6. Verify no other files are impacted
- Check all route pages to ensure they still work with updated layouts
- Ensure components don't have dependency issues (imports, types)

## Files to Create
1. `frontend/components/Navbar.tsx` - Public navigation bar
2. `frontend/components/ProtectedNavbar.tsx` - Protected (auth-aware) navigation bar
3. `frontend/components/Footer.tsx` - Site footer

## Files to Modify
1. `frontend/app/layout.tsx` - Replace header/footer with components
2. `frontend/app/(protected)/layout.tsx` - Replace header with ProtectedNavbar

## Dependencies
- Components will import from `next/link`, `next/navigation` as needed
- ProtectedNavbar will use `@/hooks/useAuth` hook
- Tailwind CSS classes preserved from original
- Font imports need to be handled (may stay in layout or move to components)

## Implementation Notes
- Preserve exact Tailwind classes and styling from originals
- Font variable setup (`Geist`, `Geist_Mono`) may need to stay in layout files since they set CSS variables used globally
- Components should be "use client" where needed (ProtectedNavbar needs it for useAuth, Navbar/Footer are static)
- Keep the responsive behavior and interactive elements identical to original
