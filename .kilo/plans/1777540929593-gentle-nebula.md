# Redesign Plan for Protected Pages

## Goal
Redesign pages in `frontend/app/(protected)/` to have a modern look with smooth edges, Dracula theme, and Claude-inspired design.

## Current State
- Layout: `frontend/app/(protected)/layout.tsx` - provides authentication checks and basic UI (header, nav).
- Pages:
  - Dashboard: `frontend/app/(protected)/dashboard/page.tsx` - shows user's comics list.
  - Comics Create: `frontend/app/(protected)/comics/create/page.tsx` (not yet viewed but exists).
  - Comics Edit: `frontend/app/(protected)/comics/[comicId]/edit/page.tsx`
  - Chapters Create: `frontend/app/(protected)/comics/[comicId]/chapters/create/page.tsx`
  - Chapters Edit: `frontend/app/(protected)/comics/[comicId]/chapters/[chapterId]/edit/page.tsx`
  - Onboarding: `frontend/app/(protected)/onboarding/page.tsx`

## Design Guidelines
- **Theme**: Dracula (dark background, purple, pink, cyan accents).
- **Smooth Edges**: Use rounded borders (rounded-xl, rounded-2xl, etc.) and possibly glassmorphism effects.
- **Claude Design**: Clean, minimal, ample whitespace, clear typography, subtle shadows, and intuitive interactions.
- **Colors**: Use Dracula palette:
  - Background: #282a36
  - Foreground: #f8f8f2
  - Accent: #bd93f9 (purple), #ff79c6 (pink), #8be9fd (cyan), #ffb86c (orange), #f1fa8c (yellow), #6272a4 (gray), #ff5555 (red)
- **Typography**: Use existing Geist fonts, but consider adjusting weights and sizes for hierarchy.
- **Components**: Cards, buttons, inputs, links should have consistent styling with hover/active states.

## Implementation Steps

### 1. Update Global Styles
- Modify `frontend/app/globals.css` to define Dracula color variables.
- Set dark mode as default (or keep system preference but lean dark).
- Add global styles for rounded elements, transitions, and glassmorphism utility classes.

### 2. Update Layout
- Adjust `frontend/app/(protected)/layout.tsx` to apply Dracula theme (maybe add a dark class to root).
- Enhance header with modern styling (glassmorphism, subtle shadow).
- Ensure navigation links and buttons follow the new theme.

### 3. Redesign Dashboard Page
- Update `frontend/app/(protected)/dashboard/page.tsx`:
  - Header section: improve user info and CTA button.
  - Comics grid: use cards with glassmorphism, hover lift effect.
  - Empty state: modern illustration or typography.
  - Loading skeletons: use animated placeholders.
  - Delete confirmation: keep but style buttons.

### 4. Redesign Comics Create/Edit Pages
- Form layout: two-column or stacked with proper spacing.
- Input fields: use updated InputField component (if exists) or create styled inputs.
- Thumbnail preview: modern upload area.
- Submit button: prominent with accent color.

### 5. Redesign Chapters Create/Edit Pages
- Similar to comics but simpler.
- Rich text editor integration (if used) should match theme.

### 6. Redesign Onboarding Page
- Welcome message, profile setup steps.
- Use cards and illustrations.

### 7. Ensure Consistency
- Use same button styles across all pages.
- Use same input styles.
- Use same card styles.
- Ensure responsive design.

### 8. Testing
- Verify all pages render correctly in dark mode.
- Check responsiveness on mobile.
- Ensure no regression in functionality.

## Notes
- We should check if there's a shared component library (e.g., `frontend/components/`) to update for consistency.
- The Dracula theme should be implemented via CSS variables so that it's easy to adjust.
- Glassmorphism can be achieved with `bg-background/50 backdrop-blur` and border.
- Smooth edges: use `rounded-xl` or `rounded-2xl` consistently.
- Shadows: use `shadow-md` and `hover:shadow-lg` for lift.

## Future Considerations
- Add a theme toggle (light/dark) if desired.
- Implement dark mode detection and persistence.
