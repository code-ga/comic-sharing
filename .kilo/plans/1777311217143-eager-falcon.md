# Login & Register UI Implementation Plan

## Overview
Build authentication UI (Login and Register pages) for the comic-sharing platform. The backend already has `better-auth` configured with email/password authentication, and the auth router is set up but has no routes. The frontend needs login/register pages and authentication state management.

## Project Structure Understanding

### Backend (Ready)
- `better-auth` configured with Drizzle adapter (PostgreSQL)
- Auth tables: `user`, `session`, `account`, `verification`
- Auth router exists at `/api/auth` prefix but has no routes defined (empty file)
- Authentication middleware already built (`authenticationMiddleware`)
- `better-auth` provides built-in API routes for auth operations

### Frontend (To Build)
- No auth pages yet - only a landing page
- Eden treaty client configured at `/api/` for type-safe API calls
- Tailwind CSS v4 with custom theme
- React 19 with Next.js 16 App Router

## API Endpoints Available (from better-auth)

Since `better-auth` has `emailAndPassword: { enabled: true }`, the following endpoints are automatically available:

- `POST /api/auth/sign-in/email` - Login with email/password
- `POST /api/auth/sign-up/email` - Register with email/password  
- `POST /api/auth/sign-out` - Logout
- `POST /api/auth/session` - Get current session
- Cookies: `session_token` (httpOnly, secure in production)

## Data Flow

1. User submits login form → call `api.auth.signIn.email` via Eden client
2. Eden client → Next.js API route `/api/auth/sign-in/email` (proxy)
3. Proxy → Backend `better-auth` handler
4. Backend validates credentials, creates session, sets httpOnly cookie
5. Cookie returned via Set-Cookie header through proxy to browser
6. Subsequent requests include cookie automatically
7. Session accessible via `api.auth.getSession`

## Key Considerations

1. **CSRF Protection**: `better-auth` handles this automatically with double-submit cookie pattern
2. **Credentials**: Eden client configured with `credentials: "include"` for cookie passing
3. **Proxy**: All `/api/*` calls go through Next.js proxy, backend host never exposed
4. **Type Safety**: Eden + TypeBox ensures compile-time type checking for all API calls
5. **Profile Creation**: Backend auth middleware requires profile for protected routes. Register flow may need to create profile after user creation.

## Validation Rules

**Login:**
- Email: Required, valid email format
- Password: Required, min 8 chars

**Register:**
- Username: Required, 3-30 chars, alphanumeric + underscores
- Email: Required, valid email format
- Password: Required, min 8 chars, at least 1 number
- Confirm Password: Must match password

## Testing Checklist

- Login with valid credentials → success, redirect, session cookie set
- Login with invalid credentials → error message
- Register with valid data → success, user created, auto-login or redirect
- Register with existing email → error message
- Password confirmation mismatch → validation error
- Session persistence across page refresh
- Logout clears session
- Protected routes redirect when not authenticated

## Success Metrics

- Login form submits and receives session cookie
- Register form creates user and profile in database
- Auth state persists across navigation
- Type-safe API calls with no TypeScript errors
- Responsive design works on mobile and desktop
- Dark/light mode compatibility
