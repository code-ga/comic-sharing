# Plan for Implementing Frontend Landing Page

## Goal
Implement the frontend landing page (`frontend/app/page.tsx`) to display comics fetched from the backend routes in `backend/src/routes/comic.ts`.

## Backend Routes to Use
From `backend/src/routes/comic.ts`, we will use:
1. `GET /comics/latest-update` - Get latest updated comics
2. `GET /comics/recently-added` - Get recently added comics
3. `GET /comics/recommended` - Get recommended comics

## Implementation Plan

### 1. Create ComicCard Component
Create a reusable component to display individual comic information:
- File: `frontend/components/ComicCard.tsx`
- Props: comic object (matching backend comic schema)
- Display:
  - Thumbnail image
  - Title
  - Optional: description preview or metadata
  - Link to comic detail page (`/comics/${id}`)

### 2. Update Landing Page (`frontend/app/page.tsx`)
Replace current content with:
- Import necessary hooks and components:
  - `useQuery` from `@tanstack/react-query`
  - `api` from `../lib/api`
  - `ComicCard` from `../components/ComicCard`
- Fetch data for three sections:
  - Latest updated comics
  - Recently added comics
  - Recommended comics
- For each section:
  - Show loading state while fetching
  - Show error state if request fails
  - Show empty state if no comics
  - Display comics in a responsive grid or scrollable list using ComicCard
- Layout:
  - Header with site title/logo
  - Three sections with titles and comic grids
  - Footer (optional)

### 3. Data Fetching Details
Use `useQuery` with:
- Query keys: `['comics', 'latest']`, `['comics', 'recent']`, `['comics', 'recommended']`
- Query functions: `api.api.comics.latestUpdate.get()`, etc.
- Handle optionalAuth (no authentication required for these endpoints)

### 4. Styling Guidelines
Follow existing Tailwind CSS patterns in the project:
- Use `flex`, `grid`, `gap`, `p-*` classes
- Maintain consistent card design with hover effects
- Use responsive classes (`sm:`, `md:`, `lg:`)

### 5. Error Handling
- Show user-friendly error messages
- Provide retry mechanism for failed requests
- Log errors to console for debugging

### 6. Loading States
- Show skeleton loaders or spinners while data is fetching
- Maintain layout stability during loading

### 7. Navigation
- Make each ComicCard clickable to navigate to comic detail page
- Use Next.js `next/link` for client-side navigation

## Files to Create/Modify
1. Create: `frontend/components/ComicCard.tsx`
2. Modify: `frontend/app/page.tsx`

## Dependencies
- `@tanstack/react-query` (already used in project)
- `next/link` for navigation
- Existing API client from `../lib/api`

## Notes
- The backend routes return comics with chapters included (based on `with: { chapters: true }` in some endpoints)
- We only need comic metadata for the landing page, not chapters
- Error responses follow the pattern from `errorResponseSchema`
- Success responses follow `baseResponseSchema` with `data` field containing the comic array

## Future Considerations
- Add pagination/lazy loading for large comic collections
- Implement actual recommendation algorithm (currently just returns latest updated)
- Add filtering/sorting options
- Implement infinite scroll for comic grids
